Point of Sale Hide Out of Stock Products

This Odoo POS module enhances the point-of-sale experience by introducing a feature to hide unavailable products based on stock availability in the linked warehouse or location. It ensures that only products with sufficient stock are visible to the POS user, streamlining the selling process and avoiding potential stock-related issues during sales.



Features
  • Simplify the POS Interface: Reduce clutter by displaying only available products.
  • Prevent Out-of-Stock Sales: Ensure the visibility of stock availability in real-time, preventing the addition of unavailable products to customer orders.
  • Support Community and Enterprise Editions

FAQ

1. Does this feature apply to all warehouses?

Can I choose which warehouse to use for stock availability checks?

The module uses the warehouse associated with the picking type configured in the POS. If no picking type is set, stock checks are skipped, and all products remain visible.

2. How does the module determine product availability?

What criteria are used to decide whether a product is available?

The module checks the product's stock quantity in the default location of the picking type linked to the POS configuration. If the quantity is greater than zero, the product is considered available.

3. What happens if a product’s stock becomes unavailable during a session?

Will the product list update automatically if stock changes during a POS session?

The product list is filtered dynamically when the POS interface is loaded or refreshed. However, real-time updates require refreshing the POS session to fetch the latest stock data.

4. What happens if the warehouse or picking type is not configured?

Will the module work if there’s no warehouse or picking type configured in the POS?

If no warehouse or picking type is set, the module defaults to showing all products without filtering for availability.
